## 2. 주요 논의

### 2-1. WebSocket 연결 구조

* **왕택준(팀장)**
  * api 연결 경로는 
  * 세션 발급 로직 위치 확인 필요성 제기

* **김경민**
  * 프론트엔드 연결 시 세션 ID 반환 방식 검토
  * WebSocket 엔드포인트 구조 제안

* **진도희**
  * 세션 생명주기 관리 방안 의견 제시

### 2-2. UI/UX 설계

* **왕택준(팀장)**
  * 화면 구성도를 기반으로 각자 담당 페이지 분배 제안
  * 로그인 플로우 설계: 웰컴 페이지 → 회원가입/로그인 → OTP 인증 → 약관 동의(소셜 사용자만)

* **김경민**
  * 시나리오 선택(DB 프롬프트 조회)과 시나리오 작성(사용자 개인화) 구분 필요성 제기
  * 대화 화면 옵션 제안: 1) 아무것도 없이 대화하는 화면 2) 말한 걸 텍스트화 시켜서 보여주는 화면

* **진도희**
  * 마이페이지 구조 제안: 히스토리 리스트 → 피드백 화면 → 히스토리 리스트
  * 피드백 화면은 대화 종료 후 즉시 노출되도록 설계

### 2-3. 디자인 컨셉

* **왕택준(팀장)**
  * 로고 색상으로 노란색, 검정색 제안
  * 헬스장(gym) 기구가 검정, 노랑이 많아서 컨셉에 맞다고 판단

* **김경민**
  * 플랫폼 정책성에 맞춰 유연하게 색상 조정 가능하다는 의견

* **진도희**
  * 원하는 색상으로 가도 되고 유지해도 된다는 의견

### 2-4. 대화 화면 및 종료 기준

* **왕택준(팀장)**
  * 대화 화면은 채팅 형태로 구성
  * 탭으로 말하기 방식 도입 제안

* **김경민**
  * 대화 종료 기준 3가지 제안:
    1. 시간 제한: 사용자 문장은 1분 이내로 말해야 함
    2. 침묵 감지: 사용자 입력 종료는 4초 이상 침묵 시
    3. 탭으로 말하기 방식
  * 1분 제한 타이머 UI 제공 필요
  * 종료 버튼 배치

* **진도희**
  * 실시간 피드백은 추후 추가하기로 합의
  * 대화 화면 옵션 중 "아무것도 없이 대화하는 화면" 선택 제안

### 2-5. 사용자 정보 및 추후 개발 계획
  
  * 성별, 나이, 직업 등 유저 정보를 AI 프롬프트에 전달 필요
  * 개인정보 화면은 현재 구현하지 않기로 결정
  * 사용자 정보 수집 항목 및 프롬프트 전달 방식 상세 설계 필요
  * 시나리오 필터링 기능은 추후 추가 예정
  * 잡음 처리는 백엔드에서 WebRTC VAD 라이브러리로 처리

### 2-6. 프론트엔드 기술 스택

* **왕택준(팀장)**
  * 빠른 프론트엔드 개발을 위해 Ant Design 대신 Tailwind CSS 사용에 동의 

* **김경민**
  * Tailwind CSS가 커스터마이징이 용이하고 빠른 개발에 적합하다는 의견
  * 빠른 프론트엔드 개발을 위해 Ant Design 대신 Tailwind CSS 사용 제안

* **진도희**
  * 빠른 프론트엔드 개발을 위해 Ant Design 대신 Tailwind CSS 사용에 동의

## 3. 결정 사항

1. **API 경로**
   * api/v1 경로 사용

2. **WebSocket 세션 관리**
   * 프론트엔드 연결 시 세션 ID 반환 구조 확정

3. **화면 플로우 확정**
   * 로그인 화면(웰컴) → 회원가입/로컬 로그인 → OTP 인증 → OTP 인증 완료 → 약관 동의(소셜 사용자만)
   * 시나리오 선택(DB 프롬프트 조회) / 시나리오 작성(프론트 생성 후 DB 저장)
   * 대화 화면(채팅 형태, 탭으로 말하기)
   * 피드백 화면(대화 종료 후 즉시 노출) → 시나리오 페이지 복귀
   * 마이페이지: 히스토리 리스트 → 피드백 화면

4. **대화 종료 조건**
   * 사용자 문장: 1분 시간 제한
   * 탭 입력 방식 도입

5. **UI 구성**
   * 탭으로 말하기 방식
   * 1분 제한 타이머 UI 제공
   * 종료 버튼 배치
   * 대화 화면은 **채팅 화면**으로 선택

5. **사용자 정보 활용**
   * 성별, 나이, 직업 등 유저 정보를 AI 프롬프트에 전달
   * 마이페이지 개인정보 수정 화면은 현재 미구현

6. **디자인**
   * 노란색, 검정색 기반 로고 및 컬러 스킴 (변경 가능)

7. **프론트엔드 CSS**
   * Tailwind CSS 사용 확정
   * Ant Design은 사용하지 않음

8. **잡음 처리**
   * 백엔드에서 WebRTC VAD 라이브러리 사용

9. **추후 개발 예정**
   * 실시간 피드백 기능
   * 시나리오 필터링 기능
   * 실시간 텍스트 변환 화면
