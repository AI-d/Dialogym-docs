
# Dialogym 요구사항 명세서 v0.2

> 프로젝트: **Dialogym – AI 기반 대화·연락 코치 플랫폼**  
> 업데이트: 2025-10-12 (KST)  
> 근거 문서: 컨셉/개요/사례분석/유저플로우
> 변경 요약: 기능(FR)/비기능(NFR)을 **MVP**와 **Should-have**로 분류. 추적 매트릭스에 우선순위 반영.

---

## 0. 문서 목적과 범위

- 이 문서는 Dialogym의 **기능(Functional)** 및 **비기능(Non‑Functional)** 요구사항을 검증 가능하게 정리한다.  
- 대상: 기획/디자인/개발/QA 전원.  
- 범위: 로그인~초안생성~피드백/재평가~히스토리~프로필 기반 변환까지 1차 MVP.

Out of Scope(1차 제외):
- 실시간 통화 중 AI 개입, 다국어 코칭, 고급 챌린지 등(개요 문서 기준).

---

## 1. 용어 정의

- **초안(Draft)**: 사용자가 선택하거나 수정 중인 메시지.
- **피드백**: 공손도/명료성/길이 등 점수+개선안.
- **재평가**: 수정된 메시지에 대한 최대 N회 재분석.
- **프로필(상대방)**: 말투/길이/관계 등 변환 파라미터.

---

## 2. 이해관계자/페르소나(요약)

- **일반 사용자**: 직장/연애/가족/면접 등 실전 메시지 코칭 필요.
- **관리자/운영자**: 정책·카테고리·시나리오 템플릿 관리.
- **QA/개발자**: 기능/성능/보안 기준을 수용시험으로 검증.

---

## 0. 분류 기준

- **MVP**: 최초 출시에서 반드시 필요. 가치 검증·핵심 플로우 성립에 필수. (P0)  
- **Should-have**: MVP 이후 단기(스프린트 2~3) 내 강화 요소. 사용자 가치↑ / 리스크↓ (P1)

---

## 1. 기능 요구사항(FR) — 분류

### ✅ MVP (P0)
- **FR-01 회원가입/로그인 (MVP)**  
  AC-01-1/2/3 유지. 이메일+비번 기반, JWT 저장.
- **FR-02 시나리오 선택 (MVP)**  
  카드 선택 및 상태 보존, 접근성 기본(키보드 선택).
- **FR-03 의도 입력 & AI 초안 3개 생성 (MVP)**  
  최소 글자수 검증, 톤 레이블과 함께 3안 표시.
- **FR-04 초안 선택 & 저장 (MVP)**  
  Draft 영속화 후 피드백 단계 전환.
- **FR-05 피드백 산출(점수/개선/수정안) (MVP)**  
  총점+세부 점수 3종, 개선 포인트 ≥2, 수정안 ≥1.
- **FR-06 수정 & 재평가 (MVP, 1회 제한)**  
  *변경*: MVP에서는 **재평가 1회**(횟수 카운트/제한 UI).
- **FR-07 최종안 저장 (MVP)**  
  Δ점수 기록 포함.
- **FR-08 히스토리 조회(요약/상세) (MVP, 축소)**  
  *축소 범위*: 최신순 정렬 + 카테고리 필터 **중 1개만** 우선.

### ⭐ Should-have (P1)
- **FR-06 재평가 2회까지 확대 (Should-have)**  
  정책/요금제 연동 시 확장.
- **FR-08 히스토리 고급 기능 (Should-have)**  
  전/후 비교 뷰 최적화, 다중 정렬·복수 필터, “다시 사용하기” 단축.
- **FR-09 상대방 프로필 등록 (Should-have)**  
  말투/길이/관계 옵션 저장.
- **FR-10 프로필 기반 메시지 변환 (Should-have)**  
  선택 프로필 스타일 반영 변환 후 Draft 생성.

---

## 2. 비기능 요구사항(NFR) — 분류

### ✅ MVP (P0)
- **NFR-P-01 성능**: 초안 생성 p95 API 응답(큐 수락) ≤ 300ms.  
- **NFR-P-02 체감 로딩**: 피드백 산출 로딩 ≤ 5초(로딩 UI).  
- **NFR-R-01 안정성**: 외부 AI 실패 3회 재시도 + 폴백(템플릿).  
- **NFR-R-02 일관성**: 최종 저장 트랜잭션 보장.  
- **NFR-S-01 인증**: 모든 API JWT 필요.  
- **NFR-S-02 암호화**: TLS 전송, 민감 데이터 저장 암호화/마스킹.  
- **NFR-O-01 로깅**: 주요 이벤트 구조화 로그.  
- **NFR-O-02 메트릭**: p95/실패율/재시도 지표 수집.

### ⭐ Should-have (P1)
- **NFR-P-03 목록 성능**: 히스토리 목록 ≤ 200ms(캐시/인덱스 최적화).  
- **NFR-S-03 권한 세분화**: RBAC(작성자/관리자 권한 분리).  
- **NFR-A11Y-01 접근성 강화**: WCAG 2.1 AA 목표 체크리스트.  
- **NFR-I18N-01 국제화 준비**: 텍스트 리소스 분리·언어팩 구조.  
- **NFR-C-01 감사/보존**: 개인정보 로그 보존 90일+ 감사 추적.

## 4. 릴리즈 체크리스트(요약)

### MVP 출시 전(P0)
- [ ] 로그인/JWT 동작, 비번 정책/오류 메시지 점검  
- [ ] 시나리오 선택 → 의도 입력 → 초안 3개 노출 경로 확인  
- [ ] 초안 저장(Draft) → 피드백 점수/개선/수정안 확인  
- [ ] 재평가 1회 제한 UI/서버 동기화  
- [ ] 최종 저장(Δ점수 포함), 히스토리 기본 조회  
- [ ] 성능 p95, 로깅/메트릭, 재시도/폴백 동작 확인

### Should-have 준비(P1)
- [ ] 재평가 상한 2회로 확장 + 정책화  
- [ ] 히스토리 전/후 비교/정렬·필터 고도화  
- [ ] 프로필 등록/변환 흐름, RBAC/접근성/국제화/감사
