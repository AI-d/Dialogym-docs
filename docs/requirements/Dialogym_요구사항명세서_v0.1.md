
# Dialogym 요구사항 명세서 v0.1

> 프로젝트: **Dialogym – AI 기반 대화·연락 코치 플랫폼**  
> 작성일: 2025-10-12 (KST)  
> 작성자: ChatGPT (초안)  
> 근거 문서: 컨셉/개요/사례분석/유저플로우

---

## 0. 문서 목적과 범위

- 이 문서는 Dialogym의 **기능(Functional)** 및 **비기능(Non‑Functional)** 요구사항을 검증 가능하게 정리한다.  
- 대상: 기획/디자인/개발/QA 전원.  
- 범위: 로그인~초안생성~피드백/재평가~히스토리~프로필 기반 변환까지 1차 MVP.

Out of Scope(1차 제외):
- 실시간 통화 중 AI 개입, 다국어 코칭, 고급 챌린지 등(개요 문서 기준).

---

## 1. 용어 정의

- **초안(Draft)**: 사용자가 선택하거나 수정 중인 메시지.
- **피드백**: 공손도/명료성/길이 등 점수+개선안.
- **재평가**: 수정된 메시지에 대한 최대 N회 재분석.
- **프로필(상대방)**: 말투/길이/관계 등 변환 파라미터.

---

## 2. 이해관계자/페르소나(요약)

- **일반 사용자**: 직장/연애/가족/면접 등 실전 메시지 코칭 필요.
- **관리자/운영자**: 정책·카테고리·시나리오 템플릿 관리.
- **QA/개발자**: 기능/성능/보안 기준을 수용시험으로 검증.

---

## 3. 유저플로우 참조(요약)

1) 로그인/회원가입 →  
2) 시나리오 선택 → 의도 입력 → AI 초안 3개 제안 → 선택 →  
3) 피드백(점수, 개선포인트, 수정안) → (선택) 수정·재평가(최대 2회) →  
4) 최종안 저장 → 히스토리 조회/전후비교 → 프로필 등록 → 프로필 기반 변환.

---

## 4. 기능 요구사항 (FR)

### FR-01 회원가입/로그인
- **설명**: 이메일/비밀번호/이름으로 회원가입, 로그인 성공 시 토큰 저장.
- **선행조건**: 비밀번호 정책(8자 이상). 이메일 형식 검증.
- **예외**: 형식 오류/인증 실패 시 에러 메시지 제공.
- **수용 기준(AC)**:  
  - AC-01-1: 유효 입력 시 회원가입 성공 후 자동 로그인 및 시나리오 선택 화면으로 이동.  
  - AC-01-2: 로그인 성공 시 JWT(또는 동등) 저장, 세션 지속.  
  - AC-01-3: 형식 오류/인증 실패 시 표준화된 에러 코드와 메시지 반환.

### FR-02 시나리오 선택
- **설명**: 카테고리 카드 목록에서 상황 선택(상사 보고/면접/연인 등).
- **AC**:  
  - AC-02-1: 카드 클릭 시 시나리오 ID가 상태/스토리지에 보존.  
  - AC-02-2: 접근성 기준에 따라 키보드로도 선택 가능.

### FR-03 의도 입력 & AI 초안 3개 생성
- **설명**: 최소 글자수(≥5) 입력 후 “AI에게 초안 받기”를 누르면 톤별(짧게/공손/따뜻) 3안 생성.
- **AC**:  
  - AC-03-1: 입력 검증(최소 글자수/금지어 필터 등) 후 요청.  
  - AC-03-2: 3개 초안이 톤 레이블과 함께 표시.  
  - AC-03-3: 초안엔 컨텍스트(시나리오, 사용자 입력)가 반영됨.

### FR-04 초안 선택 & 저장
- **설명**: 3안 중 선택 시 Draft 엔터티에 저장되고 피드백 단계로 자동 이동.
- **AC**:  
  - AC-04-1: 선택 즉시 Draft가 영속화(작성자ID, 시나리오ID, 생성시각 포함).  
  - AC-04-2: 이후 화면은 피드백 로딩 화면으로 전환.

### FR-05 피드백 산출(점수/개선/수정안)
- **설명**: 선택된 초안에 대해 AI 분석으로 총점 및 세부 점수(공손도/명료성/길이), 개선 포인트, 수정안 제공.
- **AC**:  
  - AC-05-1: 총점(100점 만점) 및 3개 항목 점수 노출.  
  - AC-05-2: 개선 포인트 최소 2개 이상.  
  - AC-05-3: 수정안 1개 이상 자동 제안.

### FR-06 수정 & 재평가(최대 2회)
- **설명**: 사용자가 텍스트 수정 후 재평가 요청 가능(상한=2회).
- **AC**:  
  - AC-06-1: 재평가 횟수 카운트/제한 UI 반영.  
  - AC-06-2: 재평가 시 새로운 점수/개선안을 반환.  
  - AC-06-3: 한계 초과 시 버튼 비활성화 및 안내.

### FR-07 최종안 저장
- **설명**: 완료 시 최종 메시지와 점수를 히스토리에 저장.
- **AC**:  
  - AC-07-1: 최초 점수 대비 변화량(Δ점수) 저장.  
  - AC-07-2: 시나리오/시간/작성자 등 메타 포함.

### FR-08 히스토리 조회/상세/전후비교
- **설명**: 필터/정렬, 상세 화면에서 초기/최종 메시지와 점수 비교, “다시 사용하기” 제공.
- **AC**:  
  - AC-08-1: 카테고리 필터, 최신순/점수순 정렬 제공.  
  - AC-08-2: 상세에서 전/후 메시지 및 점수 상승 표시.  
  - AC-08-3: “다시 사용하기” 클릭 시 새 Draft 시작.

### FR-09 상대방 프로필 등록
- **설명**: 이름/관계/말투/길이 등 상대방 스타일을 저장.
- **AC**:  
  - AC-09-1: 말투·길이·관계의 사전 정의 옵션 제공.  
  - AC-09-2: 필수 항목 검증 및 저장 성공/실패 알림.

### FR-10 프로필 기반 메시지 변환
- **설명**: 원본 메시지를 선택한 프로필 스타일로 변환하여 제시.
- **AC**:  
  - AC-10-1: 변환 결과가 프로필의 말투/길이/관계를 반영.  
  - AC-10-2: “이 메시지 사용하기”로 Draft 생성 가능.

## 5. 비기능 요구사항 (NFR)

### 5.1 성능/지연(NFR-P)
- **NFR-P-01**: 초안 생성 요청의 p95 API 응답(큐 수락 기준) ≤ 300ms.  
- **NFR-P-02**: 피드백 산출의 사용자 체감 로딩 ≤ 5초(로딩 UI/진행표시 제공).  
- **NFR-P-03**: 히스토리 목록 응답 ≤ 200ms(캐시/인덱스 적용).

### 5.2 신뢰성/가용성(NFR-R)
- **NFR-R-01**: 외부 AI API 실패 시 3회 재시도 및 폴백(사전 템플릿/메시지).  
- **NFR-R-02**: 중요 저장(최종안/점수) 트랜잭션 일관성 보장.

### 5.3 보안/프라이버시(NFR-S)
- **NFR-S-01**: 모든 API JWT(OAuth2 등 동등 수준) 인증 필수.  
- **NFR-S-02**: 전송 구간 TLS, 저장 시 민감 데이터 암호화/마스킹.  
- **NFR-S-03**: 접근제어(RBAC): 작성자/관리자만 수정·열람 권한.

### 5.4 관측성/로깅(NFR-O)
- **NFR-O-01**: 주요 이벤트(초안생성/피드백완료/저장) 구조화 로그.  
- **NFR-O-02**: p95 응답, 실패율, 재시도 카운트 메트릭 수집(Prometheus 등).

### 5.5 접근성/국제화(NFR-A11Y/I18N)
- **NFR-A11Y-01**: 키보드 내비게이션/ARIA 레이블 제공.  
- **NFR-I18N-01**: 텍스트 리소스 분리(다국어 확장 용이).

### 5.6 규정/감사(NFR-C)
- **NFR-C-01**: 개인정보 취급에 대한 로그 보존(최소 90일) 및 감사 추적.

## 6. 데이터/도메인(요약)

- **엔터티**: User, Scenario, Draft, Feedback, History, Profile(상대방).  
- **주요 필드 예시**: Draft {id, userId, scenarioId, content, createdAt}, Feedback {scores, tips, suggestion}, History {finalContent, finalScore, deltaScore}.

## 7. 시스템 제약/가정

- 외부 LLM(API) 사용을 기본 가정. 비용/쿼터 초과 시 폴백 필요.  
- 메시지 길이 상한/금지어/개인정보 필터 정책 수립.

## 9. 수용/검증 계획(샘플)

- **기능 테스트**: 스토리 기반 e2e(Playwright/Cypress), API 단위/통합.  
- **성능 테스트**: 초안/피드백 API p95 SLA 모니터링.  
- **보안 점검**: 인증/인가, 데이터 마스킹, 로그 민감정보 검증.  
- **접근성 검사**: 키보드/스크린리더 시나리오.

---

## 10. 리스크 & 완화

- **AI 지연/실패**: 큐+폴백 템플릿, 재시도/서킷브레이커, 사용자 로딩 안내.  
- **개인정보/민감 표현**: 입력 필터/경고, 보관 정책, 익명화.  
- **과도한 재평가 호출**: 횟수 제한/쿨다운, 요금 보호.
